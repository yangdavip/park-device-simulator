## 1. 项目背景与目标

### 1.1 背景

智慧园区平台开发过程中，需要大量真实感的设备上报数据用于：
- 平台功能联调测试
- 数据可视化大屏演示
- 告警规则验证
- 数据分析算法调优
- 压力测试与性能评估

真实设备部署成本高、周期长，需要一个可配置的设备数据模拟器，按合理物理逻辑生成上报数据，覆盖智慧园区全设备系统。

### 1.2 目标

| 目标 | 说明 |
|------|------|
| **全覆盖** | 覆盖智慧园区 10 大设备系统、60+ 设备类型 |
| **协议多模** | 支持 MQTT、HTTP、Modbus TCP、OPC UA 四种常见物联网协议 |
| **数据合理** | 基于物理模型和时间段规律生成数据，非纯随机 |
| **场景驱动** | 支持工作日/节假日、白天/夜间、季节温差、突发事件等场景 |
| **可配置** | 设备拓扑、上报频率、协议参数均通过配置文件管理 |
| **可扩展** | 插件式架构，新增设备类型或协议只需实现接口 |

## 3. 协议支持方案

### 3.1 协议选型

| 协议 | 适用场景 | 模拟方式 | 优先级 |
|------|----------|----------|--------|
| **MQTT** | 温湿度、能耗、环境等周期上报 | 内嵌 MQTT Broker + 设备客户端 | P0 |
| **HTTP REST** | 门禁事件、停车事件、告警上报 | HTTP POST 回调 | P0 |
| **Modbus TCP** | 电力仪表、水泵、冷水机组等工控设备 | Modbus TCP Slave 服务器 | P1 |
| **OPC UA** | 楼宇自控、高端电梯等 | OPC UA Server | P2 |

### 3.2 MQTT Topic 设计

```
park/{park_id}/{system}/{device_type}/{device_id}/properties   # 属性上报
park/{park_id}/{system}/{device_type}/{device_id}/events       # 事件上报
park/{park_id}/{system}/{device_type}/{device_id}/alarms       # 告警上报
park/{park_id}/{system}/{device_type}/{device_id}/commands     # 下行命令（模拟接收）
park/{park_id}/{system}/{device_type}/{device_id}/status      # 在线状态
```

示例：
```
park/P001/bas/ahu/AHU-F3-001/properties
→ {"ts":"2026-06-23T10:00:00+08:00","supply_temp":18.2,"return_temp":24.5,"running":true,"set_temp":24,"fan_speed":3}
```

## 4. 数据生成引擎设计

### 4.1 核心设计原则

数据不是纯随机数，而是基于**物理模型 + 时间规律 + 随机扰动**生成：

```
最终值 = 基准值(时间相关) + 物理约束修正 + 高斯随机扰动(σ)
```

### 4.2 时间规律模型

```python
# 室内温度：白天高、夜间低
base_temp = 22 + 3 * sin(2π * (hour - 6) / 24)

# 用电功率：工作日 9:00-18:00 高峰
workday_factor = 1.0 if 9 <= hour < 18 else 0.3  # 工作日
weekend_factor = 0.2  # 周末全天低

# 室外照度：日出日落曲线
lux = max(0, 50000 * sin(π * (hour - 6) / 12))  # 6:00-18:00

# CO2浓度：办公时间高、夜间低
co2 = 400 + 300 * work_activity_factor
```

### 4.3 物理约束模型

| 约束 | 规则 |
|------|------|
| 温度惯性 | 当前温度 = 上一温度 × 0.95 + 目标温度 × 0.05 |
| 功率与负载 | 电机功率 = 额定功率 × 负载率 + 空载损耗 |
| 流量守恒 | 主管流量 = Σ支管流量 |
| 电压三相平衡 | 三相电压偏差 ≤ 2%，相位差 120° |
| 充电曲线 | 电池 SOC 80% 前恒流快充，80% 后恒压慢充 |

### 4.4 场景引擎

支持以下场景模式，可通过配置切换：

| 场景 | 描述 | 效果 |
|------|------|------|
| `normal_workday` | 正常工作日 | 9:00-18:00 人流高峰，用电高峰 |
| `weekend` | 周末 | 低人流、低能耗 |
| `holiday` | 节假日 | 极低人流、最低能耗 |
| `summer_peak` | 夏季高温 | 室外 35°C+，空调满载 |
| `winter_peak` | 冬季严寒 | 室外 0°C 以下，供暖满载 |
| `fire_emergency` | 消防突发事件 | 烟感报警→消防联动→门禁全开→电梯迫降 |
| `power_outage` | 停电事件 | 电力仪表归零→备用电源启动→部分设备离线 |
| `intrusion` | 安防入侵事件 | 周界红外触发→视频联动→告警上报 |

### 4.5 告警生成规则

基于阈值+持续时间触发告警：

```yaml
alarm_rules:
  - name: "高温告警"
    condition: "return_temp > 30"
    sustain_seconds: 120
    level: "warning"
    
  - name: "烟雾报警"
    condition: "smoke_density > 50"
    sustain_seconds: 5
    level: "critical"
    
  - name: "设备离线"
    condition: "heartbeat_missed > 3"
    sustain_seconds: 0
    level: "major"
    
  - name: "电压异常"
    condition: "voltage_a < 198 or voltage_a > 242"
    sustain_seconds: 10
    level: "warning"
```

## 5. 系统架构设计

### 5.1 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 开发语言 | **Go** | 并发模型适合大量设备模拟，单二进制部署 |
| 配置格式 | **YAML** | 可读性好，支持复杂嵌套 |
| MQTT Broker | **moquitto** 或内嵌 **paho** | 轻量，标准协议 |
| Modbus | **go-modbus** 库 | 成熟稳定 |
| OPC UA | **gopcua** 库 | 官方推荐 |
| HTTP Server | **Gin** | 轻量高性能 |
| Web 管理面板 | **Vue3 + Vite** | 与既有项目技术栈一致 |
| 数据存储 | **SQLite** | 轻量，存储设备配置和运行日志 |

## 8. 关键数据模型

### 8.1 设备模型接口

```go
// Device 设备接口
type Device interface {
    // 基本信息
    ID() string
    Type() string
    System() string
    Protocol() ProtocolType
    
    // 生命周期
    Start() error
    Stop() error
    Status() DeviceStatus
    
    // 数据生成
    GenerateData(ctx ScenarioContext) map[string]interface{}
    
    // 告警检查
    CheckAlarms(data map[string]interface{}) []Alarm
    
    // 协议上报
    Report(data map[string]interface{}) error
}
```

### 8.2 场景上下文

```go
// ScenarioContext 场景上下文，传入数据生成器
type ScenarioContext struct {
    Timestamp     time.Time
    Scenario      string           // 当前场景名
    OutdoorTemp   float64          // 室外温度
    OccupancyRate float64          // 入住率 0-1
    PowerLoadRate float64          // 用电负载率 0-1
    IsWorkday     bool             // 是否工作日
    Season        string           // 季节
    ExtraParams   map[string]any   // 扩展参数
}
```
